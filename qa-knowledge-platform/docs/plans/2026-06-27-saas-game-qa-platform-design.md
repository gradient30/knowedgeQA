# SaaS 与游戏 QA 平台产品设计基线

## 目标

将现有 QA 测试知识协作平台升级为对标腾讯、网易式 SaaS 业务与游戏业务质量体系的平台。产品不做通用博客，而是服务研发、测试、QA 经理和项目负责人，沉淀质量知识、测试资产、工具经验、质量情报和验收证据。

## 行业依据

- 腾讯云产品与游戏解决方案体现出云 SaaS、游戏研发、运维和质量服务的一体化趋势。
- 网易游戏与伏羲相关公开材料显示，AI、强化学习和自动化测试已进入游戏研发与测试场景。
- 企业知识产品趋势正在从静态文档转向权限感知搜索、智能问答、Agent 工作流和治理审计。

## 产品定位

面向大厂 SaaS 与游戏研发团队的 QA 知识沉淀、工具运营、测试资产管理、质量情报与效能治理平台。

核心原则：
- 双业务域统一底座：SaaS 与游戏共享用户、权限、文件、搜索、审核和指标体系。
- 业务标签前置：每条知识、工具、资讯和测试资产都必须标注业务域、测试类型、项目阶段和适用对象。
- 先闭环后智能：V1.0 完成真实业务可用闭环，V1.1 再基于已审核数据做智能推荐、问答和 Agent 归档。

## 用户角色

- 测试工程师：创建测试经验、Bug 案例、工具评价、版本测试记录。
- 自动化测试工程师：沉淀脚本方案、CI 经验、自动化覆盖缺口和执行报告。
- QA 部门经理：审核内容、制定质量标准、查看团队效能和风险。
- SaaS 产品/项目负责人：关注 SLA、灰度、回归、线上事故和客户交付风险。
- 游戏项目负责人：关注版本提审、性能、兼容性、玩法质量、玩家反馈和本地化风险。
- 平台管理员：管理用户、权限、内容审核、资讯源和系统配置。

## 业务域设计

### SaaS 质量场景

覆盖多租户、权限、审计、SLA、API 兼容性、灰度发布、回归测试、线上故障复盘和客户交付。

必须支持的用户功能：
- SaaS 测试策略模板：范围、风险、接口、数据隔离、回归范围、上线门禁。
- API 兼容性矩阵：版本、接口、调用方、破坏性变更、验证状态。
- 灰度与回滚记录：灰度批次、监控指标、异常、决策和复盘。
- 线上事故复盘：影响范围、根因、修复、测试漏检点、预防措施。
- SLA 与质量指标看板：缺陷趋势、回归通过率、自动化覆盖率、事故数。

### 游戏质量场景

覆盖多端适配、性能帧率、弱网、玩法逻辑、数值平衡、版本提审、LQA、本地化和社区反馈。

必须支持的用户功能：
- 游戏版本质量报告：版本号、平台、渠道、核心风险、阻塞缺陷、提审结论。
- 机型兼容矩阵：设备、系统、画质档位、帧率、崩溃、发热、包体状态。
- 玩法测试清单：玩法模块、任务链路、经济系统、匹配、活动、异常路径。
- 性能压测记录：场景、人数、帧率、内存、网络、CPU/GPU、结论。
- 玩家反馈归档：来源、标签、影响范围、复现路径、关联缺陷和处理状态。

## 模块功能蓝图

### 知识库

V1.0 必须从占位页升级为可用模块：
- 内容类型：测试经验、Bug 案例、工具教程、最佳实践、SaaS 事故复盘、游戏版本报告。
- 创作能力：Markdown 编辑、结构化模板、附件证据、草稿、提交审核、发布。
- 组织能力：业务域、项目、分类、标签、可见性、作者、状态、更新时间。
- 消费能力：列表、详情、搜索、筛选、收藏、点赞、评论。

### 工具库

V1.0 支持团队运营测试工具，而不是简单链接收藏：
- 工具分类：功能、接口、性能、自动化、移动、游戏、管理、监控。
- 工具档案：名称、链接、适用业务域、使用场景、接入方式、维护人。
- 评价体系：评分、优点、缺点、适用边界、替代方案、团队推荐状态。
- 运营指标：使用次数、收藏数、评分数、推荐工具列表。

### 资讯中心

V1.0 服务 QA 经理和骨干测试工程师的信息筛选：
- 资讯源管理：名称、URL、业务域、关键词、抓取频率、启停。
- 内容处理：抓取、去重、关键词命中、相关性评分、人工审核。
- 用户消费：列表、分类、标签、收藏、分享、转为知识库草稿。

### 协作与治理

- 评论、点赞、收藏、基础通知。
- 内容审核流：草稿、待审核、已发布、驳回、归档。
- 权限：Member、Admin、SuperAdmin；Private、Team、Public 可见性。
- 审计：记录内容创建、编辑、审核、发布、删除和关键配置变更。

### 管理后台

- 用户与团队管理。
- 内容审核队列。
- 资讯源配置。
- 分类与标签治理。
- 质量运营指标：内容数、发布数、审核时长、活跃用户、工具评分、资讯命中。

## 阶段验收矩阵

| 阶段 | 范围 | 通过标准 |
| --- | --- | --- |
| P0 | 核心模块闭环 | 知识库、工具库、资讯中心不再返回“开发中”；列表、详情、创建、编辑、删除可用 |
| P1 | 业务治理 | 业务域标签、权限、审核、附件、搜索、筛选可用 |
| P2 | 协作运营 | 评论、收藏、点赞、通知、管理后台、基础指标可用 |
| P3 | 智能化 | 基于已审核内容做带来源问答、相似 Bug/文章推荐、工具推荐、资讯摘要 |

## 不做范围

V1.0 不做独立测试执行平台、不替代 Jira/禅道、不做完整自动化调度平台、不做无数据基础的 AI Agent。相关能力仅作为知识、报告、执行结果和外部链接归档。

## 成功指标

- 50 人 QA 团队可完成日常知识沉淀、工具评价、资讯筛选和版本质量归档。
- 每个核心模块至少覆盖 SaaS 与游戏两个业务域的真实模板。
- 管理者可以按业务域查看内容沉淀、审核状态和质量运营指标。
- 所有 P0/P1 功能有后端 API、前端入口、测试用例和验收记录。
