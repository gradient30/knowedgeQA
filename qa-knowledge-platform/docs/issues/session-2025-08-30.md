# 2025-08-30 问题解决会话记录

## 会话概述

本次会话解决了QA测试知识协作平台项目的Docker环境搭建和启动问题。

## 解决的问题列表

### 1. Windows环境bash脚本执行问题
- **问题**: Windows PowerShell无法直接执行bash脚本
- **解决**: 创建PowerShell版本脚本，后根据用户需求删除，保持只使用bash脚本
- **最终方案**: 使用WSL环境执行bash脚本

### 2. Poetry --no-dev参数废弃问题
- **问题**: Docker构建时Poetry报错`--no-dev`参数不存在
- **解决**: 修改Dockerfile使用`--no-root`参数分步安装
- **文件**: `qa-knowledge-platform/backend/Dockerfile.dev`

### 3. 缺少README.md文件问题
- **问题**: Poetry安装当前项目时找不到README.md
- **解决**: 创建backend/README.md文件
- **文件**: `qa-knowledge-platform/backend/README.md`

### 4. SQLAlchemy metadata属性冲突
- **问题**: Article模型中的`metadata`字段与SQLAlchemy保留属性冲突
- **解决**: 重命名为`extra_data`字段
- **文件**: `qa-knowledge-platform/backend/app/modules/knowledge/models.py`

### 5. Docker初始化脚本路径错误
- **问题**: init.sql是目录而不是文件
- **解决**: 删除目录，创建正确的SQL文件
- **文件**: `qa-knowledge-platform/backend/scripts/init.sql`

### 6. 前端autoprefixer依赖缺失
- **问题**: Next.js编译失败，找不到autoprefixer模块
- **解决**: 重新安装前端依赖
- **命令**: `docker-compose exec frontend pnpm install`
